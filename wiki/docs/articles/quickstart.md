# Tutorial 2: ブロックだけで作る /hello

「まずは動くものを返す」がテーマ。EDBB のブロックで `/hello` を作り、コード生成から実行までを一気に通します。

## ゴール

- ブロックだけで `/hello` を返す Slash コマンドを組む
- Embed 付きの返信で「できた感」を出す
- 生成コードの Token 差し込み位置を把握する

## Step 0: ワークスペースを整える

- `Blocks` レイアウトで組みやすく、`Split` レイアウトでコードも同時確認できます。好みで切り替え。
- 共有されてきたブロックがある場合はそのまま上書きしてOK。自動保存はローカルストレージに入ります。

## Step 1: 起動イベントを置く

1. 「🏁 Botが起動したとき」をワークスペースに置く。  
   ここに何も繋げなくても、生成コード側で Slash コマンドの同期が実行されます。

## Step 2: Slash コマンドと返信

1. 「⚡ スラッシュコマンド /」ブロックを追加し、`hello` と命名。  
2. 中に「↩️ 返信する」ブロックを繋ぎ、メッセージを「やっほー 👋」など好みのテキストに。

これだけで `/hello` が返る最低限の形になります。

## Step 3: Embed を足して映えさせる

1. 「✨ 新しい埋め込み(Embed)作成」を置き、そこに「設定：タイトル」「設定：説明文」「設定：色」を繋げてタイトル/説明/カラーを指定。
2. 作成した Embed ブロックを「↩️ 返信する」の入力に差し込めば、Embed 付きで返信されます。

## Step 4: コードをコピーして実行

1. 右上の「コード生成」を押して Python コードをコピー。
2. ファイルに保存し、末尾の `bot.run('TOKEN')` を自分の Bot Token で置き換え。
3. `pip install -U "discord.py[voice]"` を済ませた環境で `python bot.py`。

## 動作チェック

- 起動ログに `Synced ... command(s)` が出ているか
- Discord 側で `/hello` がサジェストされるか（出ない場合は Bot を再招待 or 数分待つ）
- 返信が Embed 付きで返るか
